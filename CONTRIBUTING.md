# Contributing to WorkSmart

Thanks for contributing. Keep changes small, reviewable, and production-safe.

## Workflow

1. Create a branch from `main` using one of:
   - `feature/<short-name>`
   - `fix/<short-name>`
   - `chore/<short-name>`
2. Make focused changes.
3. Run checks locally:
   - `npm install`
   - `npm run format:check`
4. Push branch and open a pull request.
5. Merge only after required checks pass.

## Commit style

Use clear, short commit messages with a prefix:

- `feat:`
- `fix:`
- `chore:`
- `docs:`
- `refactor:`

Example:

`chore: add release checklist and contribution guide`

## Pull request expectations

- Explain what changed and why.
- Keep scope narrow.
- Include verification steps.
- Avoid unrelated file edits.
